# تكامل قاعدة البيانات

## نظرة عامة

تم ربط التطبيق بقاعدة البيانات MongoDB لتمكين:
- إنشاء حسابات جديدة في قاعدة البيانات
- تسجيل الدخول باستخدام البيانات المحفوظة
- تخزين بيانات المستخدمين بشكل آمن

## الملفات المحدثة

### 1. صفحة التسجيل (`app/register/page.tsx`)
- **التحديثات**:
  - ربط النموذج بـ API التسجيل (`/api/auth/signup`)
  - إضافة التحقق من كلمة المرور
  - إضافة رسائل الخطأ والنجاح
  - ترجمة الواجهة للعربية
  - إضافة `HydrationBoundary` لتجنب مشاكل hydration

- **الميزات الجديدة**:
  - التحقق من تطابق كلمات المرور
  - التحقق من قوة كلمة المرور (8 أحرف على الأقل)
  - عرض متطلبات كلمة المرور
  - رسائل خطأ مفصلة

### 2. صفحة تسجيل الدخول (`app/login/page.tsx`)
- **التحديثات**:
  - ربط النموذج بـ API تسجيل الدخول (`/api/auth/login`)
  - إزالة البيانات الوهمية
  - إضافة رسائل الخطأ
  - ترجمة الواجهة للعربية

- **الميزات الجديدة**:
  - تسجيل الدخول باستخدام البريد الإلكتروني وكلمة المرور
  - التحقق من صحة البيانات
  - إضافة المستخدم تلقائياً إلى الـ store المحلي

## API Endpoints المستخدمة

### 1. تسجيل حساب جديد
```http
POST /api/auth/signup
Content-Type: application/json

{
  "name": "اسم المستخدم",
  "email": "user@example.com",
  "password": "كلمة المرور"
}
```

### 2. تسجيل الدخول
```http
POST /api/auth/login
Content-Type: application/json

{
  "email": "user@example.com",
  "password": "كلمة المرور"
}
```

## تدفق البيانات

### عند إنشاء حساب جديد:
1. المستخدم يملأ نموذج التسجيل
2. التحقق من صحة البيانات (كلمة المرور، تطابق كلمات المرور)
3. إرسال البيانات إلى `/api/auth/signup`
4. تشفير كلمة المرور وحفظها في قاعدة البيانات
5. إرجاع بيانات المستخدم (بدون كلمة المرور)
6. إضافة المستخدم إلى الـ store المحلي
7. الانتقال إلى صفحة المنتجات

### عند تسجيل الدخول:
1. المستخدم يدخل البريد الإلكتروني وكلمة المرور
2. إرسال البيانات إلى `/api/auth/login`
3. التحقق من صحة البيانات في قاعدة البيانات
4. إرجاع بيانات المستخدم
5. إضافة المستخدم إلى الـ store المحلي
6. الانتقال إلى صفحة المنتجات

## الأمان

### تشفير كلمات المرور
- استخدام `bcryptjs` لتشفير كلمات المرور
- 12 جولة تشفير (SALT_ROUNDS = 12)
- لا يتم تخزين كلمات المرور كنص عادي

### التحقق من البيانات
- استخدام Zod للتحقق من صحة البيانات
- التحقق من قوة كلمة المرور
- التحقق من صحة البريد الإلكتروني

### حماية من الأخطاء
- معالجة أخطاء قاعدة البيانات
- رسائل خطأ واضحة للمستخدم
- عدم كشف معلومات حساسة في الأخطاء
